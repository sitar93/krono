Detailed KRONO Mode Descriptions

This file describes the operation of each mode implemented in the KRONO module.

--- Implemented Modes ---

1. MODE_DEFAULT (Multiplications/Divisions)
   - Group A: Outputs clocks at multiples of the base tempo (x2, x3, x4, x5, x6 on outputs 2A-6A).
   - Group B: Outputs clocks at divisions of the base tempo (/2, /3, /4, /5, /6 on outputs 2B-6B).
   - Swap (PA1): Inverts the functions between Group A and Group B (A becomes Divisions, B becomes Multiplications).

2. MODE_EUCLIDEAN (Euclidean Rhythms)
   - Group A: Generates Euclidean rhythms based on a predefined set of steps (K) and total length (N) on outputs 2A-6A.
   - Group B: Generates Euclidean rhythms based on *another* predefined set of K/N on outputs 2B-6B.
   - Swap (PA1): Swaps the K/N parameter sets between Group A and Group B.

3. MODE_MUSICAL (Musical Ratios)
   - Generates clocks based on rhythmic ratios (fractions of the base tempo) on outputs 2-6 of each group.
   - Group A (Default): Outputs 2A-6A with ratios 1/6, 1/8, 8/1, 6/5, 4/5.
   - Group B (Default): Outputs 2B-6B with ratios 1/7, 3/4, 5/3, 7/2, 9/4.
   - Swap (PA1): Swaps the ratio sets between Group A and Group B.

4. MODE_PROBABILISTIC (Trigger Probabilities)
   - Group A: Each output (2A-6A) has a predefined increasing probability (10% to 50%) of generating a trigger on each base clock (F1).
   - Group B: Each output (2B-6B) has a predefined exponentially decreasing probability (50% to ~3%) of generating a trigger on each base clock (F1).
   - Swap (PA1): Inverts the probability curves between Group A and Group B (A becomes decreasing, B increasing).

5. MODE_SEQUENTIAL (Mathematical Sequences)
   - Group A: Outputs triggers based on a mathematical sequence (Default: Fibonacci {1, 2, 3, 5, 8}) applied to the base clock counter on outputs 2A-6A.
   - Group B: Outputs triggers based on another mathematical sequence (Default: Primes {2, 3, 5, 7, 11}) on outputs 2B-6B.
   - Swap (PA1): Swaps the sequences used between Group A (becomes Lucas {2, 1, 3, 4, 7}) and Group B (becomes Composites {4, 6, 8, 9, 10}).

6. MODE_SWING (Swing/Shuffle per Output)
   - Group A: Applies a set of swing percentages *unique to each output* (2A-6A), delaying the even beats.
   - Group B: Applies a *different* set of unique swing percentages to each output (2B-6B), delaying the even beats.
   - Swap (PA1): Swaps the sets of applied swing percentages between Group A and Group B.

7. MODE_POLYRHYTHM (X:Y Polyrhythms and Sum)
   - Group A: Outputs 2A-5A generate X:Y polyrhythms (Default: 3:2, 5:3, 7:4, 9:5). Output 6A is the sum (logical OR) of the triggers from 2A-5A.
   - Group B: Outputs 2B-5B generate *other* X:Y polyrhythms (Default: 4:3, 6:5, 8:7, 10:9). Output 6B is the sum (logical OR) of the triggers from 2B-5B.
   - Swap (PA1): Swaps the sets of X:Y polyrhythmic ratios between Group A and Group B.

8. MODE_LOGIC (Logic Gates)
   - Concept: Logically combines the calculated outputs of the Default mode.
   - Group A: Outputs the XOR result between the corresponding calculated outputs (2-6) of Default mode for A and B (e.g., 2A_logic = 2A_default XOR 2B_default).
   - Group B: Outputs the NOR result between the corresponding calculated outputs (2-6) of Default mode for A and B (e.g., 2B_logic = NOT (2A_default OR 2B_default)).
   - Swap (PA1): Inverts the logic gates between the groups (A becomes NOR, B becomes XOR).

9. MODE_PHASING (Phasing Clocks & Derived Outputs)
   - Concept: Group B has a slightly different frequency than Group A.
   - Group A (Outputs 2A-6A): Output 2A follows the base tempo. Outputs 3A-6A generate derived clocks (x2, /2, x3, /3) from the base tempo.
   - Group B (Outputs 2B-6B): Output 2B follows the deviated base tempo. Outputs 3B-6B generate derived clocks (x2, /2, x3, /3) from the *deviated* base tempo.
   - Swap (PA1): Cycles through 3 levels of frequency deviation for Group B (small, medium, large).

10. MODE_CHAOS (Lorenz Threshold Crossing)
    - Concept: Generates triggers based on the crossing of thresholds by the variables of the Lorenz chaotic system.
    - Group A (Outputs 2A-6A): Triggers based on the crossing of predefined thresholds (X_THRESHOLDS) by the X variable of the Lorenz system.
    - Group B (Outputs 2B-6B): Triggers based on the crossing of predefined thresholds (YZ_THRESHOLDS) by the Y or Z variables (selected by YZ_USE_Y).
    - Divided Triggers: A trigger is output only every 'N' threshold crossings. The value of 'N' (chaos_current_divisor) is shared among all outputs 2-6.
    - Swap (PA1): Decrements the divisor 'N' for all outputs (from CHAOS_DIVISOR_DEFAULT down to CHAOS_DIVISOR_MIN with step CHAOS_DIVISOR_STEP), wrapping back to default if the minimum is reached. The divisor state is saved/loaded.

11. MODE_BINARY (16-Step Binary Pattern Sequencer)
     - Concept: A 16-step sequencer with hardcoded binary patterns for each output.
     - Speed: The sequencer runs at 4x speed relative to the main clock (16 steps complete in 4 clock cycles of 1A/1B).
     - Output Mapping:
       * Output 2 (2A/2B): Kick
       * Output 3 (3A/3B): Snare
       * Output 4 (4A/4B): Clap
       * Output 5 (5A/5B): Open Hi-Hat
       * Output 6 (6A/6B): Closed Hi-Hat
     - Group A vs Group B: Both groups play the same drum sounds, but with slightly different patterns for variation (creates interesting polyrhythmic effects).
     - Bank Selection: 10 unique banks (0-9) with different styles:
       * Bank 0: Basic Techno
       * Bank 1: Techno variant
       * Bank 2: Industrial
       * Bank 3: Industrial variant
       * Bank 4: Trance
       * Bank 5: Trance variant
       * Bank 6: Minimal
       * Bank 7: Minimal variant
       * Bank 8: Breakbeat
       * Bank 9: Breakbeat variant
     - Bank Change: Press MOD button or send CV gate to PB4 to cycle to the next bank.
     - Bank Change Timing: The bank change is applied at the end of the current 16-step cycle (seamless transition).
     - State Saving: The current bank is saved to Flash memory via the standard mode change save procedure.
     - Swap (PA1): Cycles through the 10 banks. The current bank is saved/loaded.
